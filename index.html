2. Write code to explain prototype chaining.

– All objects in JavaScript have a prototype. An object's prototype is also considered to be an object.That prototype object has a prototype of its own, and so on until an object is reached with null as its prototype. By definition, "null" has no prototype, and acts as the final link in this prototype chain. 

        const obj1={
            fname:"srn",
            id:1504,
        }
        obj1.__proto__.name="snsnsn";
        console.log(obj1.__proto__); //{ name: 'snsnsn' }
        console.log(obj1.name); //snsnsn
                        
                        or

        const obj = {
            a: 1,
            b: 2,
            // __proto__ sets the [[Prototype]]. It's specified here as another object literal.
            __proto__: {
              b: 3,
              c: 4,
              __proto__: {
                d: 5,
              },
            },
          };
          
          // { a: 1, b: 2 } -> { b: 3, c: 4 } -> { d: 5 } -> Object.prototype -> null
          
          console.log(obj.d); // 5
          console.log(obj.e); //undefined.

4. Write a JavaScript function to retrieve all the names of object's own and inherited properties.

– To get all own properties of an object in JavaScript, we can use the Object.getOwnPropertyNames() method. This method returns an array containing all the names of the enumerable and non-enumerable own properties found directly on the object passed in as an argument.

        const user = {
            name: 'Alex',
            age: 30
        }
        const props = Object.getOwnPropertyNames(user)
        console.log(props) // [ 'name', 'age' ]

– if we want get only enumerable properties of the object, use the Object.keys() method instead:

        const user = {
            name: 'Alex',
            age: 30
        }
        const props = Object.keys(user)
        console.log(props) // [ 'name', 'age' ]
